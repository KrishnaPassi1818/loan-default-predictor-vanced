◈Borrower Segments

Unsupervised K-Means clustering (k=4) on 7 scaled numeric features (loan amount, property value, income, credit score, LTV, DTI, term), fit independently from the classifier above. These archetypes are descriptive, not predictive.

◈Model Performance Comparison
Winning model: Gradient Boosting (learning_rate=0.1, n_estimators=200), selected via GridSearchCV against ROC-AUC across 3-fold stratified cross-validation. Chart shows ROC-AUC on the held-out test set — Gradient Boosting and Random Forest were close (0.889 vs 0.887), but Gradient Boosting won on Recall too. The decision threshold was then re-tuned to 0.353 (instead of the default 0.5) to maximize F1 on the default class.
◈About RiskCompass

Project overview

RiskCompass estimates a mortgage borrower's probability of loan default and places them within a set of borrower risk archetypes discovered through clustering. Built as a summer training project covering the full ML lifecycle: data cleaning, EDA, preprocessing, unsupervised segmentation, and supervised classification.

Technology stack

LayerTechnology
FrontendCustom HTML / CSS / JavaScript
BackendFlask (Python)
ClassificationGradient Boosting (GridSearchCV-tuned)
SegmentationK-Means + PCA

DeploymentRender

Dataset

Task: Binary classification (Default / Paid) on a cleaned, class-balanced mortgage dataset, plus unsupervised borrower segmentation.
Leakage guard: rate of interest, interest rate spread, and upfront charges are dropped before training since they're only known after approval.
